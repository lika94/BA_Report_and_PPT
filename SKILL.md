# Business Analysis Deck Generator

Create investment-grade business analysis presentations in multiple professional styles. Whether you need the structured rigor of McKinsey, the strategic frameworks of BCG/Bain, or the compelling narrative of investment banking and VC pitches.

## Core Philosophy

1. **Action Title First** — Every slide title is a complete sentence stating the insight
2. **Style-Matched Structure** — Different frameworks for different contexts
3. **MECE Logic** — Mutually Exclusive, Collectively Exhaustive arguments
4. **Data as Proof** — Every claim backed by evidence
5. **Viewport Perfect** — Every slide fits exactly, no scrolling
6. **Pattern Before Case** — Abstract the business model first, then discuss specific examples
7. **System Before Design** — Distinguish business model from institutional design (bounty/matchmaking/task)
8. **Cold Start vs Long-term** — Evaluate short-term feasibility and long-term value separately, with evolution path

---

## CRITICAL: Action Title Rule

**Every slide title must be a complete sentence stating the core insight.**

### Bad Titles (Information Labels)
- ❌ "Market Analysis"
- ❌ "Competitive Landscape"
- ❌ "Financial Projections"
- ❌ "Team Overview"

### Good Titles (Action Titles)
- ✅ "The $50B market is growing at 25% CAGR driven by AI adoption"
- ✅ "We occupy the premium positioning gap between mass-market and enterprise"
- ✅ "Revenue will reach $100M by 2027 with 70% gross margins"
- ✅ "Our team has 50+ years combined experience scaling B2B SaaS"

### Title Formula
```
[Subject] + [Key Finding] + [Supporting Metric]
```

## STYLE 1: McKinsey Structure

**Best for:** Strategy presentations, board updates, complex problem-solving
**Core:** Pyramid Principle + MECE + Data-driven

### 9-Step Business Model Analysis (Integrated)

When analyzing platform business models, use this enhanced framework:

**Step 1: Pattern Definition** (Not just case description)
- What is the CORE value proposition? (Not "more options" but "making the right judgment")
- What pain point does it solve?
- How is it different from existing solutions?

**Step 2: Market Hypothesis** — Substitutes & White Space
- Current solution landscape
- Shortcomings of each approach
- Market timing judgment

**Step 3: User Analysis** — Demand-side segmentation
- User categories (by scale/industry/need/budget)
- Differentiated needs of each segment
- Early focus strategy

**Step 4: Supply-side Analysis** — Creator/service provider tiers
- Supply tiers (e.g., L1-L4 creators)
- Value positioning & monetization for each tier
- Tiered operation strategy

**Step 5: Institutional Design Comparison** (Key Insight)

Compare design options across these dimensions:

| Dimension | Task Model | Bounty Model | Matchmaking Model |
|-----------|------------|--------------|-------------------|
| Core Mechanism | Standardized tasks, creators claim | Multi-creator competition, brand selects best | Platform matches, one-on-one cooperation |
| Supply-side Launch | Easy (low barrier) | Hard (no guarantee) | Medium (need portfolio) |
| Demand-side Launch | Easy (price transparent) | Easy (low risk) | Hard (need trust) |
| Cold Start Speed | Fast | Medium | Slow |
| Market Size | Large | Medium | Small |
| Avg Deal Size | Low | High | Medium |
| Platform Take Rate | Low | Medium | High |
| Moat Depth | Shallow | Medium | Deep |
| Big Tech Threat | High | Medium | Low |

**Key Insight:** Cold start difficulty: Matchmaking > Bounty > Task
**Long-term value:** Matchmaking > Bounty > Task

**Evolution Path Template:**
```
Short-term (0-6mo): Task model for quick launch → volume
Mid-term (6-12mo): Bounty model for higher deal size → quality filter
Long-term (12-24mo): Matchmaking model for moat → data & trust barrier
```

**Step 6: Core Risks & Business Model Viability**
- Key risk identification
- Fit with existing capabilities
- Capabilities to build

**Step 7: Competitive Research** — Success & Failure Cases
- Common elements of successful platforms
- Pitfalls from failed cases
- Learnable differentiation strategies

**Step 8: Product Breakdown** — Core Features
- Demand-side core features
- Supply-side core features
- Differentiated feature design

**Step 9: Feasibility Assessment** — Resources & Gaps
- Current resource inventory
- Key gap identification
- Entry strategy & phased planning

**Step 10: Key Assumptions & Risk** — Validation Mechanism
- Key assumption list
- Validation methods & success criteria
- Plan B design

### McKinsey Deck Flow (10-15 slides)

1. **Executive Summary** — "We recommend X because of Y, Z, and W"
2. **Problem Statement** — The specific challenge/opportunity
3. **Situation Analysis** — Current state with key facts
4. **Complication** — Why status quo is untenable
5. **Resolution** — The recommended approach
6. **Supporting Logic A** — MECE argument 1 with evidence
7. **Supporting Logic B** — MECE argument 2 with evidence
8. **Supporting Logic C** — MECE argument 3 with evidence
9. **Risk Assessment** — Key risks and mitigations
10. **Implementation** — Phased plan with milestones
11. **Financial Impact** — Quantified benefits
12. **Next Steps** — Immediate actions and owners

### McKinsey Slide Anatomy

```
┌─────────────────────────────────────────┐
│  Action Title (Complete sentence)        │
├─────────────────────────────────────────┤
│                                         │
│     [Chart / Diagram / Key Visual]      │
│                                         │
│  • Supporting point 1                   │
│  • Supporting point 2                   │
│  • Supporting point 3                   │
│                                         │
│                         Source: XXX     │
└─────────────────────────────────────────┘
```

### McKinsey Frameworks Available
- **Pyramid Principle** — Top-down logic structure
- **Issue Tree** — Problem decomposition
- **Driver Tree** — Metric breakdown
- **2x2 Matrix** — Strategic positioning
- **Waterfall Chart** — Financial bridge
- **Sankey Diagram** — Flow visualization

## STYLE 2: BCG/Bain Structure

**Best for:** Market entry, growth strategy, portfolio decisions
**Core:** Strategic frameworks + Market analysis + Competitive positioning

### BCG/Bain Deck Flow (12-18 slides)

1. **Situation Assessment** — Market context and our position
2. **Market Attractiveness** — Size, growth, profitability
3. **Competitive Landscape** — Key players and dynamics
4. **Strategic Options** — 3 distinct paths evaluated
5. **Option A Analysis** — Pros, cons, requirements
6. **Option B Analysis** — Pros, cons, requirements
7. **Option C Analysis** — Pros, cons, requirements
8. **Recommended Strategy** — Chosen path with rationale
9. **BCG Matrix** — Portfolio positioning (if applicable)
10. **Value Creation Plan** — How we win
11. **Capability Requirements** — What we need to build/buy
12. **Financial Projections** — 5-year outlook
13. **Implementation Roadmap** — Phased approach
14. **Risk Management** — Key risks and responses
15. **Investment Requirements** — Capital and resources needed

### BCG/Bain Frameworks Available
- **BCG Growth-Share Matrix** — Stars, Cash Cows, Question Marks, Dogs
- **Experience Curve** — Cost vs. cumulative volume
- **Competitive Advantage Map** — Cost vs. differentiation
- **Market Segmentation** — Attractive segments matrix
- **Capability Heat Map** — Build/buy/partner decisions

## STYLE 3: Investment Banking Structure

**Best for:** M&A pitches, capital raising, sell-side/buy-side advisory
**Core:** Transaction rationale + Valuation + Market context

### IB Pitch Deck Flow (15-25 slides)

**Section A: Market Context**
1. **Executive Summary** — Transaction thesis in 3 bullets
2. **Industry Overview** — Market size, growth, trends
3. **Competitive Landscape** — Key players and positioning
4. **Market Dynamics** — Drivers and disruptors

**Section B: Company/Asset Profile**
5. **Business Overview** — What they do, how they make money
6. **Financial Performance** — Revenue, EBITDA, margins
7. **Strategic Position** — Competitive advantages
8. **Growth Opportunities** — Upside potential

**Section C: Transaction Rationale**
9. **Strategic Logic** — Why this deal makes sense
10. **Synergy Analysis** — Cost and revenue synergies
11. **Value Creation** — How value will be unlocked

**Section D: Valuation**
12. **Valuation Summary** — Multiple methodologies
13. **Comparable Company Analysis** — Trading multiples
14. **Precedent Transactions** — M&A comps
15. **DCF Analysis** — Discounted cash flow
16. **Football Field** — Valuation range summary

**Section E: Execution**
17. **Process Overview** — Timeline and milestones
18. **Buyer Universe** — Strategic and financial buyers
19. **Key Considerations** — Risks and mitigation
20. **Why Us** — Our track record and capabilities

### IB Frameworks Available
- **Football Field** — Valuation range visualization
- **Accretion/Dilution** — EPS impact analysis
- **Sources & Uses** — Deal financing breakdown
- **Pro Forma** — Combined financials
- **Synergy Bridge** — Value creation waterfall

## STYLE 4: VC Pitch Structure

**Best for:** Startup fundraising, Series A/B/C
**Core:** Vision + Traction + Team + Market

### VC Pitch Deck Flow (10-12 slides)

1. **Vision** — The future you're creating (one powerful sentence)
2. **Problem** — Pain point that's big, growing, urgent
3. **Solution** — Your product and how it solves the problem
4. **Why Now** — Market timing and tailwinds
5. **Market Size** — TAM/SAM/SOM with growth rate
6. **Product** — Demo/screenshots + key features
7. **Traction** — Metrics that prove product-market fit
8. **Business Model** — How you make money + unit economics
9. **Competition** — 2x2 positioning matrix
10. **Team** — Key players + relevant wins
11. **Financials** — 3-5 year projections + use of funds
12. **Ask** — Specific amount + what it achieves

### VC Pitch Principles
- **Hook in 30 seconds** — First slide must grab attention
- **Show, don't tell** — Demo > description
- **Metrics that matter** — Revenue, growth rate, retention, LTV/CAC
- **Competitive moat** — What makes you defensible
- **Clear ask** — Exact amount and milestones

## Phase 1: Research & Framework Application

### Research Strategy by Style

**McKinsey:**
- Search for: Industry reports, academic studies, expert interviews
- Focus: Rigor, methodology, comprehensive analysis
- Sources: Gartner, McKinsey Global Institute, academic journals

**BCG/Bain:**
- Search for: Market sizing, competitive dynamics, strategic positioning
- Focus: Market structure, growth drivers, competitive advantage
- Sources: BCG reports, industry associations, company filings

**Investment Banking:**
- Search for: Comparable companies, precedent transactions, market multiples
- Focus: Valuation, transaction comps, financial metrics
- Sources: Capital IQ, PitchBook, company 10-Ks

**VC Pitch:**
- Search for: Market trends, startup benchmarks, growth rates
- Focus: Market opportunity, traction metrics, competitive landscape
- Sources: Crunchbase, PitchBook, industry blogs

## Error Prevention Checklist

**Before Analysis:**
1. **Pattern Abstraction** — What is the generic business model behind this case?
2. **Constraint Confirmation** — What limitations has the user emphasized before?
3. **Core Question** — What is the key question for this business model? (Network effects? Unit economics? Moat?)
4. **Value Proposition** — Who does the platform help solve what problem? Not "more options" but "making the right judgment"

**During Analysis:**
5. **Self-check after each section** — "What key business model question does this section answer?"
6. **Distinguish model from design** — Same business model can have different institutional designs (bounty/matchmaking/task)
7. **Separate cold start from long-term** — Evaluate short-term feasibility and long-term value separately

**Common Error Patterns to Avoid:**

| Error | Manifestation | Prevention |
|-------|--------------|------------|
| **Concrete Dependency** | Using specific cases instead of abstract patterns, treating validation tools as the business model itself | Ask before analyzing: "What pattern does this case represent?" |
| **Form Over Substance** | Framework completed but core questions not addressed | Self-check: "What key business model question does this answer?" |
| **Instruction Loss** | User's previous emphasis not internalized in current analysis | Review user history before analysis, confirm constraints |
| **Speed Over Quality** | Rushing to complete analysis without confirming core questions | Output core judgment first, then expand details |

**After Analysis:**
8. **Error Induction** — If errors occur, induct root causes (tabular format)
9. **Update Skill** — Add new insights and error patterns to this skill
10. **Archive Conversation** — Save to project memory for future reference
